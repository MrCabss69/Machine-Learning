Preparaci√≥n de los datos - Preprocesamiento¬∂

Ya conocemos la estructura de nuestros datos, ahora toca currarselo y prepararlos para el modelo. Esta parte es muy importante, aunque consigamos encontrar la arquitectura perfecta para la tarea a resolver, si los datos de entrenamiento no est√°n bien, el modelo no va a aprender.

  1. Data Cleaning: localizaci√≥n y correci√≥n errores o discrepancias en los datos
  2. Data Integration: fusi√≥n de informaci√≥n extra√≠da de m√∫ltiples fuentes para delinear y crear un conjunto de datos √∫nico
  3. Data Transformation: normalizaci√≥n, estandarizaci√≥n, discretizaci√≥n..
  4. Data Reduction: si el tama√±o del dataset es muy grande, hay que reducir el tama√±o del conjunto de datos manteniendo informaci√≥n crucial.
  5. Identificar y manejar los valores faltantes.
  6. Manejar outliers: Z-score o IQR
  7. Splitting the dataset en 3 conjuntos: entrenamiento (70), test(20) y validaci√≥n (10)

Enfoque Naive¬∂

Como podemos ver hay una correlaci√≥n lineal entre el √°rea y el precio de una casa, un enfoque b√°sico ser√≠a intentar hallar la l√≠nea recta que minimiza la suma de las distancias cuadr√°ticas (errores cuadr√°ticos) de todos los registros. Queremos encontrar los par√°metros √≥ptimos de la ecuaci√≥n de una recta $y = bx + a + \epsilon $ o, en t√©rminos de par√°metros, $$y = \theta_1 \cdot x + \theta_0 + \epsilon$$

Vectorialmente, esta relaci√≥n lineal se puede representar como $y = \mathbf{\theta}^T \cdot [x, 1]$, donde $\mathbf{\theta} = [\theta_1, \theta_0]^T$ es el vector de par√°metros y $[x, 1]$ representa el vector de caracter√≠sticas ampliado con un t√©rmino constante para incorporar $\theta_0$

In¬†[¬†]:

X_mean = np.mean(train_df['area'])                                              # caracter√≠sticas con las que predecimos

Ahora, ¬øC√≥mo funciona realmente el ML?¬∂

El primer paso es seleccionar una arquitectura de modelo adecuada que pueda abordar el problema en cuesti√≥n, que no es tarea sencilla. Una vez establecida esta arquitectura, procedemos a entrenar el modelo utilizando el conjunto de datos de entrenamiento.

El proceso de entrenamiento se caracteriza por una reducci√≥n iterativa de una m√©trica conocida como 'valor de p√©rdida'. El modelo recibe un conjunto de entradas que representan las caracter√≠sticas de un registro concreto. Estas entradas se procesan secuencialmente a trav√©s de las capas del modelo, acabando en una predicci√≥n. A continuaci√≥n, la predicci√≥n se compara con el valor real, y esta comparaci√≥n da lugar a una estimaci√≥n de la p√©rdida, que mide cu√°n lejos estuvo la predicci√≥n del valor real.

Bas√°ndose en esta p√©rdida, el modelo ajusta internamente los pesos de sus capas, con el objetivo de mejorar las predicciones en futuras iteraciones. Lo vemos en m√°s profundidad en el pr√≥ximo tutorial, ¬°espero que os haya gustado y hay√°is aprendido algo nuevo!
